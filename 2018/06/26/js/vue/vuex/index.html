Vuex 的设计之美

2018-06-25
vuex

Vuex 是一个专为 Vue.js 应用程序开发的状态管理工具。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。Vuex 灵感来自 Flux 和 Redux，青出于蓝，更上一层楼。

相关教程

  • Vuex 官网

1. 准备工作

一般使用 vue-cli v3 搭建脚手架时勾选 vuex 即可
vue-cli v2 需要手动下载依赖

2. 仓库（Store）

也可以叫数据容器，用来放置项目的所有共享数据。
每个项目只允许有一个仓库（多个也可以，不推荐酱紫干，难维护，复杂度高）。

3. 状态（State）

也可以叫数据，共享数据，只是视图（页面显示的数据）都是因 State 的变化而变化，所以称它为“状态”（State）
获取方式比较直接 store.state.[name]，不能直接设置值，比如 store.state.name = 'Super'

mapState 辅助函数

放于组件 computed 函数中

当命名相同时，可直接使用数据形式

或者

4. 获取状态（Getter）

一般用在获取那些需要把数据（State）修饰过的数据。

在组件中使用

  1. this.$store.getters.surname
  2. this.$store.getters.fullName
  3. this.$store.getters.getFullName('Upper')
mapGetters 辅助函数

放于组件 computed 函数中

5. 突变（Mutation）

前面说过修改 State 不能直接以赋值的方式，Vuex 这里修改 State 需要提交一个突变（Mutation），以便于记录状态的变化过程。
格式：一个字符串的 事件类型（type），一个 回调函数（handle），回调用来修改状态
「 突变必须是同步函数，否则异步函数里状态变更无效 」

在组件中调用

  1. this.$store.commit('happy new year')
  2. this.$store.commit('rename', '素伯')
  3. this.$store.commit({ type: 'changeOfAge', age: 18 })
mapMutations 辅助函数

放于组件中 methods 方法中

6. 动作（Action）

与 Mutation 类型，但它提交的是 Mutation，支持异步操作
「 与 React 中的 Redux 主要区别，Redux 没有这个概念，复杂度飙升 」

在组件中使用

  1. this.$store.dispatch('happy new year')
  2. this.$store.dispatch('rename', '素伯')
  3. this.$store.dispatch({ type: 'changeOfAge', age: 18 })
mapActions 辅助函数

放于组件中 methods 方法中，与 Mutation 一毛一样

7. 模块（Module）

由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿。

局部与全局状态
  1. Mutation 与 Getter 接收第一个参数是局部 State
  2. 对于 Action 的 context 中暴露的 state 为局部，rootState、rootGetters 为全局
  3. Getter 第二参数为局部 getters，第三个参数 rootState 为全局
命名空间
  • 如果在模块中不指定命名空间，那么 getter、dispatch、commit 不受局部影响，调用时当全局调用，如：this.$store.commit('moduleSuper: happy new year')
  • 如在 modules 中加上：namespaced: true 指明开启命名空间，那么调用就得带上命名空间
  • 如：this.$store.Super.commit('happy new year')
  • 在命名空间里声明全局 Action，在 actions 方法中加上 root: true 即可
